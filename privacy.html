Privacy

How Liaison Keyboard handles user data.

Last updated: March 19, 2026. This summary reflects the currently shipped product surfaces, including authentication, billing, generated replies, and saved conversation history.

1. What we process

Liaison Keyboard processes account details needed for authentication, subscription status, and support. When you use the product, message context is sent to the generation service so reply suggestions can be created for your selected persona.

2. Saved conversation data

The current product includes saved threads and conversation history for signed-in users. That means the service is not a strict zero-retention system. We store the data needed to support chat history, plan entitlements, and product operations, and we limit access to the authenticated user scope used by the app.

3. Billing and infrastructure

Subscription checkout and billing management are handled through Stripe. Authentication, product data, and server-side functions run through the app stack described in the current product brief, including user-scoped access controls for account data.

4. Requests and support

If you need help with privacy, billing, or account data, email support@liaisonkeyboard.com. We use support conversations only to address the request you submit.
