Privacy

How Liaison Keyboard handles user data.

Last updated: March 25, 2026. This policy covers both the marketing site and the app experience.

What we process

Liaison Keyboard processes account details, billing metadata, submitted message context, and product telemetry necessary to operate the service. Generated replies are assistive output, not automated sending behavior.

Full Memory is opt-in

Full Memory is a privacy-first opt-in feature. When enabled, it lets Liaison Keyboard retain conversational context inside isolated memory containers so the user can maintain continuity across threads.

Thread isolation

Memory containers are designed to prevent one user's conversation context from leaking into another. Thread linking is treated as a user-directed action rather than a default behavior.

Billing and infrastructure

Stripe handles checkout, invoices, payment methods, and subscription management. The app stack handles authentication, user-scoped data access, and generation workflows.
